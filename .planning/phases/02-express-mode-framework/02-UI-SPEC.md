# Phase 2 -- UI Design Contract

> Visual and interaction contract for Phase 2: Express Mode Framework.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope

**This phase has no new frontend views or components.**

The discuss-phase (CONTEXT.md D-01) descoped all portal express UI. Requirements EXPRESS-10 (kanban), EXPRESS-11 (detail views), and EXPRESS-12 (artifact viewer) are DESCOPED. Express projects are transient workflows with no portal presence.

All Phase 2 deliverables are backend-only: DB models (IntakeSession, ExpressMetric), MCP tools (session continuity, manifest sync, express metrics), skill file updates (orchestrator startup, intake routing), and Alembic migrations.

**No existing portal views are modified.** The kanban, project cards, and project detail pages remain unchanged.

The design contract below documents the established design system for reference by downstream consumers (planner, executor) who may need to add informational UI elements such as toast notifications, inline alerts, or status indicators as part of backend integration work.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` (--bg-primary) | Page background, table background |
| Secondary (30%) | `#1a1f2e` (--bg-card) | Cards, modals, dropdown menus, form controls |
| Accent (10%) | `#FF9900` (--lcars-amber) | See reserved-for list below |
| Destructive | `#ff8787` (--lcars-red) | Destructive action labels only |

Accent `#FF9900` reserved for:
- Primary button background
- Masthead bottom border (2px)
- Card left border (3px, on hover escalates to `--lcars-amber-hover`)
- Active/current nav link text and left border
- Active tab underline
- Expanded accordion left border
- Modal top border (3px)
- Drawer panel left border
- Spinner color
- Sorted table column indicator
- Switch toggle (checked state)
- Form control focus border

Additional semantic colors in the established palette:
- `#73bcf7` (--lcars-blue): Links, breadcrumb links, secondary interactive text
- `#9966CC` (--lcars-purple): Available for phase-specific status indicators
- `#69db7c` (--lcars-green): Success states, "content active" card indicator
- `#5cb85c` / `#e8a838` / `#c9190b` (score-green/amber/red): Score/health indicators

Source: `lcars-theme.css` `:root` block lines 10-43

## Copywriting Contract

Phase 2 has no new user-facing UI screens. The copywriting contract covers potential backend-triggered feedback elements (toast notifications, alerts, error messages) that may surface in the existing portal.

| Element | Copy |
|---------|------|
| Primary CTA | not applicable -- no new UI screens |
| MCP sync success (toast, if shown) | "Project synced to portal" |
| MCP sync failure (alert, if shown) | "Portal sync failed. Your local manifest is safe. Retry by saving again." |
| MCP unavailable (inline alert) | "Portal connection unavailable. Session continuity, express mode, and portal project discovery are disabled. Check your API key configuration." |
| Express mode disabled (intake hint) | "Express mode requires a portal connection. Configure your API key to enable it." |
| Empty project list (portal query) | "No projects found for your email. Start a new project with /rhdp-publishing-house, or check that your email is configured correctly." |
| Destructive confirmation | not applicable -- no destructive actions in this phase |

Note: These are skill-level text strings displayed in the Claude Code terminal, not portal UI copy. The portal UI itself is unchanged this phase.

Source: CONTEXT.md D-06 (MCP unavailability behavior), D-05 (project discovery hint), D-07 (email identification)

## Accessibility Notes

No new accessibility requirements for Phase 2 (no new UI). Existing PatternFly components provide built-in ARIA attributes and keyboard navigation. The LCARS theme maintains sufficient color contrast:
- Body text `#d2d2d2` on `#0f1117` background: contrast ratio 11.6:1 (AAA)
- Muted text `#888` on `#0f1117` background: contrast ratio 5.9:1 (AA)
- Amber accent `#FF9900` on `#0f1117` background: contrast ratio 7.8:1 (AAA)
