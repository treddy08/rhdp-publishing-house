```yaml
---
# MCP external Route (only /mcp path exposed, API key auth at application level)
{% if mcp_route_host is defined and mcp_route_host %}
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: {{ app_name }}-mcp
  labels:
    app: {{ app_name }}
    component: mcp
  annotations:
    haproxy.router.openshift.io/timeout: 180s
spec:
  host: "{{ mcp_route_host }}"
  path: /mcp
  to:
    kind: Service
    name: {{ app_name }}-backend
  port:
    targetPort: {{ backend_port }}
  tls:
    termination: edge
    insecureEdgeTerminationPolicy: Redirect
{% endif %}
```

Notes:

- Route targets the backend service (not frontend) on `backend_port`
- Route path is `/mcp` -- only MCP traffic is routed externally
- Secret uses `stringData` (not `data`) following existing pattern (no base64 encoding needed)
- The `dict2items` filter converts the `mcp_api_keys` dict to a list of `{key, value}` pairs for iteration

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External HTTPS -> MCP Route | Edge-terminated TLS. Only /mcp path exposed. API key auth at application level. |
| Volume mount -> Backend pod | K8s Secret mounted as read-only file with 0400 permissions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-19 | Information Disclosure | manifests-infra.yaml.j2 Secret | mitigate | Secret uses `stringData` (rendered by K8s). API keys are SHA-256 hashed in the Secret, never plaintext. Volume mount has `defaultMode: 0400`. |
| T-01-20 | Spoofing | MCP Route | mitigate | Route exposes only `/mcp` path. API key auth enforced by ApiKeyAuth middleware at application level. Edge TLS termination prevents MITM. |
| T-01-21 | Tampering | dev.yml / dev.yml.example | accept | Vars files are gitignored (never pushed to remote). Example file contains placeholder values only. |
| T-01-22 | Information Disclosure | RCARS_INTERNAL_URL env var | accept | Cluster-internal service DNS. Not a secret. Exposed as env var for configurability. |
</threat_model>
