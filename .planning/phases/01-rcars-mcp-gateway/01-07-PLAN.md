**1. docs/architecture/rcars-integration.md** -- System architecture document covering:

  ```
  Claude Code User --> HTTPS --> MCP Route (ph-mcp) --> PH Backend (FastAPI)

  ```
  Include the full diagram from the RESEARCH.md architecture section.
- **Auth Model**: Two boundaries:
  1. External: API key auth (SHA-256 hashed keys, hmac.compare_digest, volume-mounted Secret)
  2. Internal: SA token auth (auto-mounted K8s token, TokenReview validation, allowlist check)
- **Network Topology**: Cross-namespace call from `publishing-house-dev` to `rcars-dev` via ClusterIP service DNS
- **Deployment Components Table**: Which repo, what changes, managed by whom
- **Data Flow**: Step-by-step for a typical `ph_rcars_query` call from Claude Code to RCARS and back

**2. docs/admin/mcp-auth.md** -- API key lifecycle admin guide covering:

- **Overview**: MCP endpoint uses API key auth for external Claude Code users
- **Key Generation**: Step-by-step with exact commands:
  1. `openssl rand -hex 32` to generate raw key
  2. `echo -n "<raw-key>" | sha256sum | awk '{print $1}'` to hash
  3. Add hashed value to `ansible/vars/dev.yml` under `mcp_api_keys` dict (key name: user identifier, value: SHA-256 hash)
  4. Run deployer: `cd rhdp-publishing-house-portal/ansible && ansible-playbook deploy.yml -e @vars/dev.yml`
  5. Give raw key to user (never share the hash)
- **Key Revocation**: Remove entry from vars, redeploy, pod restarts (D-01)
- **Key Storage**: Volume-mounted K8s Secret at `/etc/ph/mcp-api-keys/keys.yaml`. File format:
  ```yaml
  user-name: "sha256:<hex-digest>"
  ```
- **Troubleshooting**: Common issues (key file not found, auth failures, pod not restarted after key change)

**3. docs/admin/rcars-service-auth.md** -- RCARS SA auth admin guide covering:

- **Overview**: PH backend authenticates to RCARS using its K8s ServiceAccount token
- **SA Allowlist Configuration**: How to add/remove SAs from `sa_allowlist` in RCARS Ansible vars
  - Format: `system:serviceaccount:<namespace>:<sa-name>`
  - Current PH SA: `system:serviceaccount:publishing-house-dev:default`
- **Deployment**: `cd rcars-advisory/ansible && ansible-playbook deploy.yml -e @vars/dev.yml`
- **Cross-Namespace DNS**: Service URL pattern (`rcars-api.rcars-dev.svc.cluster.local:8080`)
- **Verification**: Commands to verify connectivity:
  ```
  oc exec deployment/ph-dashboard-backend -n publishing-house-dev -- \
    curl -s http://rcars-api.rcars-dev.svc.cluster.local:8080/api/v1/health
  ```
- **NetworkPolicy Checks**: Commands to check for blocking policies
- **Troubleshooting**: Token rotation, allowlist mismatches, DNS resolution failures
