**New auth.py contents:**

```python
"""MCP API key authentication middleware.

Validates API keys on every tool call using SHA-256 hash comparison.
Keys are loaded from a volume-mounted K8s Secret YAML file at startup (D-01).
"""
from __future__ import annotations

import hashlib
import hmac
import logging
from pathlib import Path

import yaml
from fastmcp.server.middleware import Middleware, MiddlewareContext
from fastmcp.server.dependencies import get_http_headers
from fastmcp.exceptions import ToolError

logger = logging.getLogger(__name__)


class ApiKeyAuth(Middleware):
    """Validate API key on every tool call via SHA-256 hash comparison.

    Keys are read from a YAML file with format:
        key-name: "sha256:<hex-digest>"

    The key-name is for admin bookkeeping only (D-02).
    """

    def __init__(self, keys_file: Path):
        self._keys_file = keys_file
        self._valid_hashes: dict[str, str] = {}
        self._load_keys()

    def _load_keys(self) -> None:
        """Read YAML map of key-name -> sha256-hash from volume-mounted Secret."""
        if not self._keys_file.exists():
            logger.warning("API key file not found at %s — no keys loaded", self._keys_file)
            return
        with open(self._keys_file) as f:
            data = yaml.safe_load(f) or {}
        self._valid_hashes = {
            name: h.removeprefix("sha256:") for name, h in data.items()
        }
        logger.info("Loaded %d API key(s) from %s", len(self._valid_hashes), self._keys_file)

    def _verify_key(self, raw_key: str) -> bool:
        """Hash the incoming key and compare against stored hashes (timing-safe)."""
        incoming_hash = hashlib.sha256(raw_key.encode()).hexdigest()
        return any(
            hmac.compare_digest(incoming_hash, stored_hash)
            for stored_hash in self._valid_hashes.values()
        )

    async def on_call_tool(self, context: MiddlewareContext, call_next):
        """Intercept every tool call to validate API key."""
        # CRITICAL: include={"authorization"} because get_http_headers()
        # strips the authorization header by default (FastMCP 3.2+ behavior)
        headers = get_http_headers(include={"authorization"}) or {}
        auth_header = headers.get("authorization", "")

        if not auth_header.startswith("Bearer "):
            raise ToolError("Authentication required: missing or invalid Authorization header")

        raw_key = auth_header.removeprefix("Bearer ").strip()
        if not self._verify_key(raw_key):
            raise ToolError("Authentication failed: invalid API key")

        return await call_next(context)
```

MCP tools are registered by importing tool modules (tools.py, rcars_tools.py).
API key auth is enforced via ApiKeyAuth middleware when a key file is configured.

import logging

from fastmcp import FastMCP

from app.core.config import settings

# Build middleware list -- API key auth when key file is configured (D-01, D-03)

mcp = FastMCP(
    name="publishing-house",

        "launch instructions, validation result storage, and RCARS content advisory "
        "tools for RHDP content lifecycle projects."

)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Spoofing | auth.py API key validation | mitigate | SHA-256 hash comparison via `hmac.compare_digest` prevents timing attacks. Keys stored as hashes, never plaintext. |
| T-01-11 | Information Disclosure | auth.py error messages | mitigate | ToolError messages say "invalid API key" -- do not reveal which key was tried or how many keys exist. Never log raw API key values. |
| T-01-12 | Information Disclosure | server.py key file path | accept | Log file path for debugging but never log file contents. File has 0400 permissions via K8s Secret defaultMode. |
| T-01-13 | Elevation of Privilege | auth.py on_call_tool | mitigate | Middleware intercepts ALL tool calls via `on_call_tool` hook. No tool can bypass auth. This is enforced at the FastMCP framework level. |
| T-01-14 | Denial of Service | auth.py key validation | accept | SHA-256 hash + compare per request is negligible CPU. No rate limiting in this phase -- acceptable for <20 internal users. |
| T-01-09 | Tampering | config.py default URL | accept | Default `rcars_internal_url` is cluster-internal DNS. Overridable via env var. In-cluster traffic is not encrypted but is network-policy-controlled. Acceptable for internal service communication. |
</threat_model>
