**Module structure:**

```python
"""RCARS API client service.

Async HTTP client for RCARS v2 with SA token auth, exponential backoff
retry, and structured error handling.
"""
from __future__ import annotations

import asyncio
import logging
from pathlib import Path

import httpx

logger = logging.getLogger(__name__)

SA_TOKEN_PATH = Path("/var/run/secrets/kubernetes.io/serviceaccount/token")
```

**RCARSError exception class:**
```python
class RCARSError(Exception):
    """Custom exception for RCARS API errors."""
    def __init__(self, message: str, status_code: int | None = None, retryable: bool = False):
        super().__init__(message)
        self.status_code = status_code
        self.retryable = retryable
```

**RCARSClient class with these methods:**

- `__init__(self, base_url: str, timeout: float = 30.0)`: Store `_base_url` (stripped of trailing `/`) and `_timeout`. Does NOT import or depend on `config.py` -- callers provide the URL.

- `_read_sa_token(self) -> str | None`: Read SA token from `SA_TOKEN_PATH` on every call (RCARS-01: never cache). Return `None` with `logger.warning` on `FileNotFoundError`.

- `async _request(self, method: str, path: str, *, json: dict | None = None, params: dict | None = None, timeout: float | None = None, max_retries: int = 3) -> dict`: Core request method per D-04:
  - Build full URL from `_base_url + path`
  - Retry loop: 3 attempts with delays [1, 2, 4] seconds (exponential backoff)
  - On each attempt: read SA token fresh, set `Authorization: Bearer <token>` header if token exists
  - Create `httpx.AsyncClient(timeout=timeout or self._timeout)` per request
  - On `response.status_code < 400`: return `response.json()`
  - On `400 <= status < 500`: raise `RCARSError` immediately with `retryable=False` (fail fast on 4xx per D-04)
  - On `500+`: set `last_error`, continue retry loop
  - On `httpx.ConnectError`: set `last_error`, continue retry loop
  - On `httpx.TimeoutException`: set `last_error`, continue retry loop
  - Between retries: `logger.warning("RCARS request failed (attempt %d/%d), retrying in %ds: %s", ...)`
  - After all retries exhausted: raise `RCARSError(f"RCARS {method} {path} failed after {max_retries} retries: {last_error}", retryable=False)` (D-05 moderate verbosity)

- `async query_advisor(self, query: str, poll_interval: float = 3.0, timeout: float = 120.0) -> dict`:
  - POST to `/api/v1/advisor/query` with body `{"query": query, "stages": ["prod", "dev", "event"], "include_zt": True}`
  - Extract `job_id` from response
  - Poll loop: GET `/api/v1/advisor/query/{job_id}/result` every `poll_interval` seconds
  - If `result["status"]` is `"completed"` or `"failed"`: return result
  - After `timeout` seconds: return `{"status": "timeout", "error": f"Advisor query timed out after {timeout}s", "result": None}`

- `async catalog_search(self, query: str | None = None, limit: int = 20, offset: int = 0) -> dict`:
  - GET `/api/v1/catalog` with params `{"limit": limit, "offset": offset}`
  - Return response dict

- `async catalog_item(self, ci_name: str) -> dict`:
  - GET `/api/v1/catalog/{ci_name}`
  - Return response dict

- `async health(self) -> dict`:
  - GET `/api/v1/health` with `max_retries=1` and `timeout=5.0`
  - On any exception: return `{"status": "unavailable", "error": str(e)}`
  - On success: return response dict
