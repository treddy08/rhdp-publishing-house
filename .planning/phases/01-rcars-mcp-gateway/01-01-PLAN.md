3. Add a helper function `_parse_sa_allowlist(allowlist_str: str) -> set[str]` that splits the comma-separated string into a set, stripping whitespace:
```python
def _parse_sa_allowlist(allowlist_str: str) -> set[str]:
    if not allowlist_str:
        return set()
    return {sa.strip() for sa in allowlist_str.split(",") if sa.strip()}
```

4. Add an async function `_validate_sa_token(token: str, allowlist: set[str]) -> str | None` that:
   - POSTs to `https://kubernetes.default.svc/apis/authentication.k8s.io/v1/tokenreviews` with the token
   - Uses the pod's own SA token from `/var/run/secrets/kubernetes.io/serviceaccount/token` for auth
   - Uses the CA cert from `/var/run/secrets/kubernetes.io/serviceaccount/ca.crt` for TLS verification
   - Checks `response["status"]["authenticated"]` is True
   - Extracts `response["status"]["user"]["username"]` (format: `system:serviceaccount:<namespace>:<name>`)
   - Checks the username is in the allowlist set
   - Returns the username string on success, None on failure
   - Catches all exceptions (httpx errors, JSON errors, KeyError) and logs a warning, returns None
   - Uses `httpx.AsyncClient` with a 5-second timeout

5. Modify `get_current_user` to become `async def get_current_user` and add the SA token check BEFORE the X-Forwarded-Email check:
```python
async def get_current_user(request: Request) -> str:
    settings = request.app.state.settings
    if settings.dev_user:
        return settings.dev_user

    # Check for SA token auth (Bearer token from service accounts)
    auth_header = request.headers.get("authorization", "")
    if auth_header.startswith("Bearer "):
        token = auth_header.removeprefix("Bearer ").strip()
        allowlist = _parse_sa_allowlist(settings.sa_allowlist_str)
        if allowlist:
            sa_identity = await _validate_sa_token(token, allowlist)
            if sa_identity:
                return sa_identity

    # Fallback to OAuth proxy headers
    email = request.headers.get("X-Forwarded-Email", "")
    if not email:
        email = request.headers.get("X-Forwarded-User", "")
    return email
```

6. Update `require_auth` to become `async def require_auth` since it now awaits `get_current_user`:
```python
async def require_auth(request: Request) -> str:

    if not user:

```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PH Backend -> RCARS API | Cross-namespace call with SA token. PH is semi-trusted (same cluster, different namespace). |
| K8s TokenReview API | RCARS validates SA tokens by calling K8s API. TokenReview is the trusted authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | auth.py _validate_sa_token | mitigate | Validate SA token via K8s TokenReview API (not self-parsed JWT). Check username against explicit allowlist. |
| T-01-02 | Information Disclosure | auth.py logging | mitigate | Never log raw SA tokens. Log only SA identity string on successful validation. |
| T-01-03 | Elevation of Privilege | auth.py SA allowlist | mitigate | Allowlist is explicit (comma-separated in env var). Only listed SAs bypass OAuth proxy auth. Empty allowlist means SA auth is disabled. |
| T-01-04 | Tampering | manifests-app.yaml.j2 | accept | SA allowlist value set via Ansible vars (gitignored). Ansible deployer is the only change path per D-08. Low risk -- same trust level as other env vars. |
| T-01-05 | Denial of Service | _validate_sa_token | mitigate | 5-second timeout on TokenReview API call. On failure, falls through to email auth (does not block the request pipeline). |
</threat_model>
